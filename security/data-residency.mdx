## Regional Deployment

HANA's infrastructure is deployed regionally. When you select a region, all data processing â€” AI reasoning, patient interactions, clinical notes, audit logs â€” stays within that region.

This is an architectural guarantee, not a policy promise. The systems are physically separate.

### Region Selection

Region is configured at the organization level and cannot be changed after deployment:

```javascript
const hana = new HanaClient({
  apiKey: process.env.HANA_API_KEY,
  region: 'eu-west' // Data stays in EU-West (Ireland)
});
```

<Warning>
Region cannot be changed after initial deployment without a data migration process. Choose carefully during onboarding.
</Warning>

### What Does NOT Cross Regions

- Patient data never crosses regional boundaries
- AI model inference runs on regional compute
- No data is sent to third-party AI providers (OpenAI, Anthropic, etc.)
- Backup replication stays within the same geographic region

## Data Retention

Default retention periods:

| Data Type | Retention |
|-----------|----------|
| Conversation transcripts | 7 years |
| Clinical notes | 7 years |
| Audit logs | 7 years |
| Voice recordings | 90 days (configurable) |
| Patient memory/preferences | Until patient deletion or org offboarding |

Retention periods are configurable per organization. Data deletion is available via API or dashboard.

### Right to Erasure (GDPR)

For GDPR regions, patient data can be fully deleted:

```bash
DELETE /v1/patients/{patient_id}?erase=true
```

This removes all patient data including:
- Demographics and clinical data
- Engagement history and transcripts
- Clinical notes
- Memory and preference data
- Audit logs (anonymized, not deleted, per regulatory requirement)
